    furo-custom-method #

    @furo/furo-data v2.0.0-rc.5
    import '@furo/furo-data/src/furo-custom-method.js';
    exports <furo-custom-method> custom-element-definition
    superclass LitElement
    mixes FBP
    summary interface component to handle custom methods

    furo-custom-method is a interface component to handle custom methods.

    Attributes and Properties #

    service #

    service

    Name of the service.

    method #

    method

    Name of the custom method / rel.

    Events #

    hts-updated #

    @-hts-updated → HTS

    Fired when hts was updated by ƒ-hts-in.

    request-aborted #

    @-request-aborted → Request

    Fired when a request was canceled.

    request-started #

    @-request-started → Request

    Fired when a request is sent.

    response-raw #

    @-response-raw → Object

    Fired when a response is received.

    response-error #

    @-response-error → Object

    Fired when an error has occoured. This is a general error event. The specific error events are fired additionally.

    response-error-[status-code] #

    @-response-error-[status-code] → Object

    Fired when an error has occoured. This is a specific error event.

    fatal-error #

    @-fatal-error → Request

    Requests are made via the Fetch API if possible.Fallback XMLHttpRequest

    response-error-4xx #

    @-response-error-4xx → Object

    Fired when an error has occoured. This is a group error event. E.g. response-error-5xx, response-error-4xx

    response-error-5xx #

    @-response-error-5xx → Object

    Fired when an error has occoured. This is a group error event. E.g. response-error-5xx, response-error-4xx

    response-error-raw #

    @-response-error-raw → Object

    Fired when a error has occoured.

    response #

    @-response → Object

    Fired when a response is received.

    Methods #

    updateQp #

    updateQp(qp `` key Object ) ⟹ void

    `` Object → ƒ-update-qp

    Update query params a qp like {“active”:true} will just update the qp active

    If the current value of the qp is not the same like the injected value, a qp-changed event will be fired

    • qp
    • key value pairs

    bindRequestData #

    bindRequestData(dataObject `` ) ⟹ void

    `` → ƒ-bind-request-data

    Binds a furo-data-object type.

    • dataObject

    clearQp #

    clearQp() ⟹ void

    * → ƒ-clear-qp

    clear the query params that you have setted before



    trigger #

    trigger() ⟹ void

    * → ƒ-trigger

    trigger the method with respect for binded-requset-object



    triggerEmpty #

    triggerEmpty() ⟹ void

    * → ƒ-trigger-empty



    triggerWithBody #

    triggerWithBody(body `` ) ⟹ void

    `` → ƒ-trigger-with-body

    trigger the method with data

    • body

    htsIn #

    htsIn(hts `` ) ⟹ void

    `` → ƒ-hts-in

    • hts

    abortPendingRequest #

    abortPendingRequest() ⟹ void

    * → ƒ-abort-pending-request
